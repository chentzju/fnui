FNUI 2.0 概述

蜂鸟UI(以下称FNUI) 2.0 是基于FNUI 1.0版本的基础上，针对框架过重、加载过多、 模板引擎过于复杂等因素，重新构造的新版现代浏览器框架。

框架重写了核心内容，同时参考bootstrap和amaze ui的控件书写规范，定义了新的组件和控件，并使用自己的命名空间fn。

FNUI 2.0 采用requirejs管理模块，实现模块的自动加载，使用less管理样式表模块，实现样式可复制性； 同时FNUI2.0使用grunt进行发布，兼容requirejs和普通加载两种加载模式，开发者可以根据自己的工作习惯 使用FNUI的功能。

yeoman脚手架能够为开发者生成基于FNUI的基础框架和结构，并且内嵌了grunt，能轻松实现web服务和版本发布。

直接引用

你也可以通过直接将FNUI的文件引用到页面的方式引入FNUI 2.0。

通过link标签引用CSS样式表

<link href = "your path/fnui.min.css" rel="stylesheet">

如果你还想使用FNUI的js功能

<script src = "your path/fnui.min.js"> </script>

如果你是AMD规范和模块化加载的拥护者，你还可以使用requirejs来加载，此时，你的代码看起来应该是这样的

<script data-main="your main file path/main" src = "your path/require.js"> </script>

当然，如果你觉得requirejs太重了，你可以使用FNUI团队专门为AMD模块化加载支持而开发的fnuiloader， 为了减轻重量，fnuiloader不支持data-main入口，所以，你要单独引入你的main文件，不过即使是这样，加载量 也从原来的80KB下降到2KB了

<script src = "your path/fnuiloader.min.js"> </script>

<script src = "your path/main.js"> </script>

使用yeoman脚手架

使用yeoman脚手架构建，能够实现一键自动化搭建fnui 2.0前端目录，并且自动加载内嵌依赖，自动生成grunt文件， 可以用来实现web服务器、实时刷新、打包发布等功能

安装node.js

开发者可以根据node.js官网的安装指引获取并安装node.js，也可以在ftp://109.5.13.213/6-Software/NODEJS/获取node

安装grunt-cli和yeoman

grunt-cli能确保在cmd命令行中运行grunt,安装yeoman确保我们能正常使用yo命令。通过node.js的包管理器npm来安装即可，-g表示安装为全局依赖。

安装grunt-cli

npm install -g grunt-cli

安装yeoman

npm install -g yo

如果你是离线环境，可以到ftp://109.5.13.213/6-Software/NODEJS/获取node.js、grunt-cli和yo。 安装完node.js后，将grunt-cli和yo解压缩到在C:\\用户目录\AppData\Roaming\npm下

安装脚手架工具

请确保将FNUI的脚手架工具下载到本地，并且链接到全局依赖中。

cd your-fnui-generator-path/generator-fnui

添加链接

npm link

如果没有联网，也可以直接将generator-fnui文件夹放到C:\\用户目录\AppData\Roaming\npm\node_modules

运行脚手架

通过运行脚手架，并且跟着脚手架指引进行，就可以构建一个FNUI 2.0的前端。

yo fnui

如果你能看到fnui脚手架的欢迎页面，说明你已经成功安装了fnui的yeoman脚手架，一起来yo吧！

当项目代码完全生成后，在项目路径下，运行命令，就可以在本地的9002端口上开始一个web服务器，该服务器支持实时刷新，便于实时修正和加载前端的样式。

grunt server

至此，你已经使用yeoman脚手架生成了你的项目，你可以对你的项目进行自主开发了。
